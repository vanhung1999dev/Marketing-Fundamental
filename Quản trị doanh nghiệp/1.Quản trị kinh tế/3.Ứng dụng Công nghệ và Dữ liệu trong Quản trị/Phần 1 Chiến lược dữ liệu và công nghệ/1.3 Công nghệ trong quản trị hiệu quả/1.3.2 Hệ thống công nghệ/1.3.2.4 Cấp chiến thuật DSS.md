### Hệ Thống DSS — Cấp Chiến Thuật

DSS (Decision Support System) là lớp analytics và intelligence phía trên các operational systems — biến data thành insights để support decisions ở cấp tactical và strategic.

> **Bài học:** Operational systems (ERP, CRM, HRMS) capture data. DSS translates data into decisions. Nếu không có DSS layer, managers swimming in data nhưng thirsting for insight. Nếu không có good operational systems, DSS có nothing useful to analyze.

---

**DSS Pyramid:**

```
DESCRIPTIVE (What happened?)
  → Reports, dashboards, KPI monitoring
  → "Doanh thu tháng này là bao nhiêu?"
  → Tools: Power BI, Tableau, Looker, Google Looker Studio

DIAGNOSTIC (Why did it happen?)
  → Drill-down analysis, root cause exploration
  → "Tại sao doanh thu khu vực HN giảm 15%?"
  → Tools: BI with ad-hoc querying, Excel pivot

PREDICTIVE (What will happen?)
  → Forecasting, trend modeling, ML
  → "Doanh thu Q4 sẽ là bao nhiêu?"
  → Tools: Python/R, Azure ML, Google Vertex AI

PRESCRIPTIVE (What should we do?)
  → Optimization, recommendation engines
  → "Pricing nào sẽ maximize margin?"
  → Tools: Advanced ML, optimization algorithms
```

Most VN SME should focus on Descriptive and Diagnostic first. Predictive and Prescriptive require data maturity and talent not yet available.

**Dashboard Design Principles:**

```
Principle 1: ONE page, ONE decision
  → Each dashboard should support one type of decision
  → Sales dashboard: "Is pipeline healthy for next month?"
  → Finance dashboard: "Are we on track for month-end close?"

Principle 2: Lead with answer, support with data
  → "Revenue is 12% below target" (clear statement)
  → Then: breakdown by product, region, rep
  → Don't make user hunt for the conclusion

Principle 3: 3–5 key metrics, not 20
  → Cognitive overload = no action
  → If everything highlighted, nothing is highlighted

Principle 4: Context always
  → "Revenue: 8.5 tỷ" → meaningless without context
  → "Revenue: 8.5 tỷ (target 10 tỷ, -15%; vs same period last year: -3%)"

Principle 5: Right refresh rate
  → CEO weekly dashboard: refresh weekly
  → Operations: refresh daily
  → Don't build real-time if decisions are made weekly
```

**DSS Implementation Journey:**

```
Stage 1 (Foundation): Connect sources to BI tool
  → ERP data → Power BI → P&L dashboard
  → CRM data → Pipeline dashboard
  → Value: Replace manual Excel reporting; same data, faster

Stage 2 (Insights): Root cause analysis capability
  → Drill-down: click on revenue miss → see by product, region, rep
  → Trending: 6-month charts for all KPIs
  → Value: Managers can diagnose problems themselves

Stage 3 (Foresight): Predictive capability
  → Sales forecasting from pipeline data
  → Churn prediction from customer behavior
  → Value: Proactive management vs reactive

Stage 4 (Recommendation): Prescriptive analytics
  → Dynamic pricing recommendations
  → Inventory optimization
  → Value: Human judgment augmented by algorithms
```

> **Phân tích sâu:** Forrester research: data-driven companies grow 30% faster than those relying on intuition alone. Nhưng "data-driven" is not about having dashboards — it's about having a decision culture where data is used to challenge assumptions, test hypotheses, và track interventions. A company where every manager reviews their dashboard weekly and acts on it is far more data-driven than one with a beautiful dashboard nobody opens.

> **Sai lầm phổ biến #1:** Build DSS before fixing source systems. "Let's build a beautiful dashboard." → Dashboard shows wrong numbers because ERP data is dirty → nobody trusts it → dashboard abandoned. Fix: SSOT first, then reporting.

> **Sai lầm phổ biến #2:** Create too many dashboards. 50 dashboards = 0 dashboards (nobody knows which to use). Better: 3–5 core dashboards (CEO, Sales, Finance, Operations), reviewed in fixed cadence, updated regularly. More dashboards → more maintenance burden → staleness → distrust.

> **Cạm bẫy:** Confuse activity with insight. Dashboard shows 100 metrics updating in real-time. Manager spends hours watching numbers change but doesn't know what to do differently. DSS must connect to decisions: "When this metric goes below threshold X, here is what we do." Without decision connection, DSS is entertainment, not management tool.
