### Hệ Thống ERP — Cấp Chiến Thuật

ERP (Enterprise Resource Planning) là hệ thống tích hợp toàn bộ các luồng tài nguyên chính của doanh nghiệp — tài chính, sản xuất, chuỗi cung ứng, mua hàng, bán hàng — trong một nền tảng dữ liệu thống nhất.

> **Bài học:** ERP không phải phần mềm kế toán mở rộng — là xương sống vận hành của công ty. Khi ERP hoạt động đúng, mọi quyết định (tài chính, vận hành, chiến lược) đều được inform bởi real-time, accurate data. Khi ERP fail to implement, nó là một trong những expensive mistakes trong business.

**Core ERP Modules:**

| Module | Chức năng | Liên kết với |
|---|---|---|
| **Financial Management** | GL, AP, AR, Fixed Assets | Mọi module khác |
| **Procurement** | PO, vendor management, receiving | Finance, Inventory |
| **Inventory Management** | Stock levels, movements, valuation | Sales, Production |
| **Sales Management** | Order processing, shipping, invoicing | Finance, Inventory |
| **Production** | BOM, work orders, capacity planning | Inventory, Finance |
| **HR & Payroll** | Headcount, time, pay | Finance |
| **Reporting** | Consolidated management reports | All modules |

**ERP Implementation — critical success factors:**

```
BEFORE Implementation:
  □ Document current processes (AS-IS) — what do you do now?
  □ Design future processes (TO-BE) — how should it work in ERP?
  □ Identify gaps: which TO-BE cannot be done in standard ERP?
  □ Data cleanup: cleanse existing data before migration
  □ Executive sponsor: C-suite committed, not just IT project

DURING Implementation:
  □ Phased rollout — one module or department at a time
  □ Parallel run — run old and new system for 1–3 months
  □ Key user training — train the trainer approach
  □ Data migration testing — validate migrated data before go-live

AFTER Go-Live:
  □ Hypercare period — extra support for first 30 days
  □ Process audit — are people actually following new processes?
  □ Report setup — build management reports from ERP data
  □ Decommission old systems — don't let shadow systems persist
```

**ERP vs Best-of-Breed:**

| Approach | Pros | Cons | Best when |
|---|---|---|---|
| **ERP (integrated)** | Single data source; less integration work; one vendor | Less specialized in each area; can be rigid | Processes are standard; want simplicity |
| **Best-of-breed** | Best-in-class each function; more flexible | Integration complexity; data silos risk | Competitive advantage from specialized tool |

**Hybrid approach (common in practice):** ERP for finance/ops core + CRM specialist (HubSpot) + HRMS specialist (Base.vn) + integration layer connecting them.

> **Phân tích sâu:** Gartner research: average ERP implementation costs 2–3× initial budget estimate and takes 2× planned timeline. Why: scope creep (adding features mid-project), data quality issues discovered late, change resistance, and underestimating process redesign complexity. Success predictor: companies that invest in process design BEFORE selecting technology have significantly better outcomes.

> **Sai lầm phổ biến #1:** Choose ERP based on brand/price alone without demo. Every ERP has different strengths. Demo với actual business scenarios: "Show me how I create a purchase order, receive goods, and match to invoice." The one that feels most natural for your team's workflow is often the right choice.

> **Sai lầm phổ biến #2:** Underestimate implementation effort. "We'll implement in 3 months." Mid-market ERP implementations typically take 6–18 months. Underresourcing → rushed → poor adoption → ERP becomes expensive data entry system rather than decision support tool.

> **Cạm bẫy:** Over-customize ERP to match current (broken) processes. Every customization: (a) increases implementation cost, (b) makes future upgrades harder, (c) may be locking in a bad process. Before customizing: ask "Is our current process actually a best practice, or is it just how we've always done it?" Sometimes the ERP is right and your process needs to change.
