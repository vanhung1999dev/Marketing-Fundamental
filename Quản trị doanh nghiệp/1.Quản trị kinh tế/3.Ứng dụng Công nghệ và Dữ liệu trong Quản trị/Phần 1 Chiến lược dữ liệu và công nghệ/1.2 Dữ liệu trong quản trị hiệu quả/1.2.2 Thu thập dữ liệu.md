### Thu Thập Dữ Liệu

Data collection là điểm khởi đầu của mọi data-driven decision. Nếu data không được thu thập đúng — đầy đủ, chính xác, nhất quán, và kịp thời — mọi phân tích phía sau đều unreliable dù tool tốt đến đâu.

> **Bài học:** Garbage in, garbage out. Đầu tư vào data collection process returns nhiều hơn đầu tư vào fancy analytics tools khi data quality kém. Fix the collection first; build the analysis second.

---

**Structured vs Unstructured Data:**

| | Structured | Unstructured |
|---|---|---|
| Định nghĩa | Dữ liệu có schema cố định, dạng bảng | Dữ liệu không có cấu trúc trước định |
| Ví dụ | ERP transactions, spreadsheets, SQL tables | Emails, customer reviews, voice, images |
| % của dữ liệu doanh nghiệp | ~20% | ~80% |
| Dễ phân tích | Cao | Thấp (cần NLP, Computer Vision, ML) |
| VN SME priority | Focus here first — highest ROI | Advanced capability — defer until Stage 3 |

---

**Data Collection Methods:**

**Automated (ưu tiên nhất):**
- System-generated: ERP logs every transaction tự động
- API integration: CRM pull data từ email system
- IoT/Sensor: machine data, GPS tracking, temperature
- POS system: mỗi sale được captured automatically

**Semi-automated:**
- Barcode/QR scanning: inventory receiving tại kho
- Mobile forms: field sales reporting qua app
- OCR: scan invoices và extract structured data

**Manual (hạn chế cho critical metrics):**
- Customer surveys (NPS, satisfaction)
- Market research interviews
- Physical inventory count (stocktake)

Priority: Automate everything possible at point of transaction. Manual collection là expensive, slow, và error-prone.

---

**Data Quality Dimensions:**

| Dimension | Definition | Common failure in VN SME |
|---|---|---|
| **Accuracy** | Data reflects reality | Wrong price entered in ERP |
| **Completeness** | No missing critical values | Customer phone number blank |
| **Consistency** | Same data same across systems | CRM name ≠ ERP name for same customer |
| **Timeliness** | Available when needed | Sales data T+3 days instead of T+1 |
| **Uniqueness** | No duplicates | Same customer entered twice in CRM |
| **Validity** | Data in expected format/range | Date entered as "tháng 3" not "03/2024" |

**Nghị định 13/2023/NĐ-CP — Bảo vệ Dữ liệu Cá nhân VN:**

Có hiệu lực từ 1/7/2023:

```
Key requirements:
  □ Consent: phải có sự đồng ý trước khi thu thập dữ liệu cá nhân
  □ Purpose limitation: chỉ dùng data cho mục đích đã thông báo
  □ Data minimization: chỉ thu thập data thực sự cần thiết
  □ Security: phải có biện pháp bảo mật phù hợp với rủi ro
  □ Breach notification: báo cáo vi phạm trong 72 giờ
  □ Data subject rights: người dùng có quyền truy cập, sửa, xóa data

Penalty: Vi phạm có thể bị phạt đến 100 triệu VNĐ và xử lý hình sự

Implications cho SME:
  - E-commerce: privacy policy rõ ràng; opt-in cho marketing emails
  - B2B: business contact data có scope khác consumer data
  - CRM: document basis for holding và processing customer data
```

**Data Governance — ai owns quality:**

```
Data Owner: Business unit head
  → Accountable for data quality in their domain
  → Signs off on definitions and acceptable quality thresholds

Data Steward: Day-to-day caretaker
  → Enforces standards, monitors quality
  → Escalates systemic issues to Data Owner

Data Consumer: Anyone using the data
  → Reports issues when data looks wrong
  → Does not fix without going through Steward/Owner

Rules:
  □ Every dataset has a named Data Owner
  □ Data definition documented before collection starts
  □ Automated quality checks where possible
  □ Monthly data quality scorecard reviewed by Data Owners
```

> **Phân tích sâu:** Amazon's "Data as Product" mindset: every internal data source is treated like a product with a customer (internal teams who use it), SLAs (quality, freshness, availability), và an owner accountable for those SLAs. This prevents silos và ensures teams investing in collecting data also invest in maintaining quality. Contrast with "data dump" approach: collect everything, quality nobody's problem, downstream users deal with mess.

> **Sai lầm phổ biến #1:** Collect data without a use case. "Chúng ta nên thu thập thêm data." Thu thập cái gì? Để trả lời câu hỏi nào? Data storage is cheap; data processing và quality maintenance là không. Every data point collected creates ongoing responsibility. Start với: "What decision do I need to make? What data would help?"

> **Sai lầm phổ biến #2:** Rely on manual data collection cho critical metrics. Manual collection creates: inconsistency (people define differently), delay (collected periodically), errors (human data entry), và gaps (missed when person is absent). For any metric critical to business decisions → find automated collection method.

> **Cạm bẫy:** Over-survey customers. Survey every touchpoint → survey fatigue → response rate drops → biased sample (chỉ most engaged respond). Selective, purposeful surveying at key moments (post-purchase, post-support) với short surveys gets better response rates. NPS: 1–2 questions. CSAT: 1 question. Detailed feedback: 5–10 questions maximum.
