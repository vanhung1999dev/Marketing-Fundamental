### Hệ Thống Giám Sát và Dashboard Vốn Lưu Động

Dashboard vốn lưu động chuyển hóa dữ liệu thô từ nhiều nguồn — AR aging, inventory levels, bank balances, AP schedule — thành bức tranh tổng thể dễ hiểu, kịp thời, có thể hành động. Thiết kế đúng dashboard giúp CEO và CFO nhìn thấy rủi ro trước khi chúng trở thành khủng hoảng.

> **Bài học:** Dashboard tốt trả lời một câu hỏi: "What do I need to ACT on today?" Không phải "What happened last month?" — mà là "What is at risk right now and what should I do?" Mọi metric trên dashboard phải có action trigger rõ ràng — không có trigger = không nên trên dashboard.

---

**Dashboard Design Principles:**

```
1. ACTIONABLE: Every metric has a clear action when it goes red
2. TIMELY: Data no older than 24–48 hours for critical metrics
3. LAYERED: Executive summary (1 page) → drilldown (department level) → detail (transaction level)
4. COMPARATIVE: vs Target + vs Prior Period + vs Industry benchmark
5. FORWARD-LOOKING: Include 4–8 week forecast, not just current state
```

**Technology Stack for WC Dashboard:**

```
Option 1: Simple (SME starter):
  Excel / Google Sheets:
    Data pull: manual or semi-automated from accounting export
    Dashboard: pivot tables + conditional formatting (traffic lights)
    Update: weekly (finance team builds and updates)
    Cost: near zero; adequate for <50 tỷ revenue businesses
    
Option 2: Intermediate:
  MISA / Fast Accounting built-in reports + Google Data Studio
    Data pull: automated from accounting system
    Dashboard: auto-refreshed with Google Data Studio/Looker Studio (free)
    Update: daily
    Cost: accounting system license (already paying); GDS free
    
Option 3: Advanced:
  BI Tool (Power BI, Tableau) connected to ERP:
    Data pull: automated; real-time
    Dashboard: interactive; drilldown; multi-dimensional
    Update: near real-time
    Cost: 500–2,000 USD/month depending on tool and scale
    
Recommendation: Start with Option 1 (Excel); upgrade to Option 2 when tracking 
                becomes burden; consider Option 3 only at 500+ tỷ revenue
```

> **Phân tích sâu:** Jeff Bezos famously ran Amazon on a system of "metrics that matter" — a small, carefully chosen set of leading indicators that he reviewed weekly. For WC management, the equivalent is 5–8 metrics that tell the health of the whole system: cash position, 4-week cash forecast, DSO, DIO, DPO, overdue AR%, and inventory aging. These 7 numbers tell 90% of the WC story. Everything else is supporting detail. Resist the temptation to build 50-metric dashboards that nobody reads.

> **Sai lầm phổ biến #1:** Build a great dashboard but review it monthly. Dashboard review frequency should match decision-making frequency: executive summary weekly; treasury detail daily; full operational review monthly. A weekly dashboard reviewed monthly is just a monthly report in different formatting.

> **Sai lầm phổ biến #2:** Use lagging indicators only (what happened last month). Effective WC management requires leading indicators: 4-week cash forecast (forward-looking), order pipeline (future AR), purchase order schedule (future inventory/AP). Mix of lagging (what happened) + coincident (what is now) + leading (what will happen) gives full picture.

> **Cạm bẫy:** Define all metrics as green because the targets were set too loose. "DSO 70 days: 🟢" — because target was set at 75 days (vs industry best practice of 45). Dashboard traffic lights are only meaningful if targets are calibrated to best-in-class or company strategy, not to current performance (that just shows "we're doing as well as we always did").
