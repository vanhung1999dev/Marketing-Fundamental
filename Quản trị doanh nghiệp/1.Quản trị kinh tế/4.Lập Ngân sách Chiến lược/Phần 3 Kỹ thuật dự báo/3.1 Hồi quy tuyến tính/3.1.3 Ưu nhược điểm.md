### Ưu Nhược Điểm Của Hồi Quy Tuyến Tính Trong Dự Báo Ngân Sách

Hồi quy tuyến tính là công cụ mạnh nhưng không phải vạn năng. Hiểu rõ ưu và nhược điểm giúp biết khi nào nên dùng, khi nào nên dùng phương pháp khác, và khi nào nên kết hợp nhiều phương pháp.

> **Bài học:** Không có công cụ dự báo hoàn hảo. Người giỏi không phải là người biết công cụ mạnh nhất — là người biết khi nào dùng công cụ nào, và quan trọng hơn, biết limitations của công cụ mình đang dùng để không over-confidence vào forecast của mình.

---

**Ưu điểm của Hồi Quy Tuyến Tính:**

```
1. INTERPRETABLE:
   "For every 1% increase in GDP, revenue increases 0.45 tỷ"
   → Management can understand and challenge the model
   → Black box models (neural networks) cannot provide this

2. QUANTIFIES RELATIONSHIPS:
   Provides precise estimates of how much X influences Y
   Useful for sensitivity analysis: "If GDP drops 2%, impact on revenue is..."

3. IDENTIFIES KEY DRIVERS:
   Multiple regression reveals which factors actually explain Y
   And which factors LOOK related but aren't significant after controlling for others

4. CONFIDENCE INTERVALS:
   Provides statistical uncertainty measure — not just a point estimate
   "Revenue expected to be 117 tỷ ± 9 tỷ at 95% confidence"

5. WIDELY AVAILABLE:
   Built into Excel (Data Analysis ToolPak), freely accessible
   No specialized software needed for basic applications

6. ENABLES SCENARIO ANALYSIS:
   Change assumption for X → immediately recalculate Y
   "If GDP grows only 3% instead of 6%, revenue forecast becomes..."
```

---

**Nhược điểm và Giới Hạn:**

```
1. LINEARITY ASSUMPTION (biggest limitation):
   Assumes Y-X relationship is straight-line
   Many real business relationships are non-linear:
     Marketing diminishing returns
     Network effects (non-linear growth)
     Threshold effects (nothing happens until a certain point)
   Forcing these into linear model → poor predictions

2. STATIONARITY REQUIRED:
   Model assumes the relationship between X and Y is stable over time
   If market structure changes (new competitor, tech disruption),
   model trained on old data becomes unreliable
   "The past relationship no longer holds"

3. EXTRAPOLATION DANGER:
   Reliable only within historical range of X values
   Forecasting with X values outside training range = extrapolation
   Extrapolation errors can be very large

4. MULTICOLLINEARITY (multiple regression):
   When X predictors correlate with each other, coefficients become unstable
   Example: GDP and consumer confidence both rise together → hard to separate their effects
   Solution: remove one correlated predictor; use PCA; use regularization

5. REQUIRES SUFFICIENT HISTORICAL DATA:
   Rule of thumb: min 20–30 data points per predictor
   Small samples → wide confidence intervals → low precision forecasts
   Many VN SMEs have limited historical data → this is a real constraint

6. DOES NOT CAPTURE AUTOCORRELATION:
   Time series data has autocorrelation (this month predicts next month)
   Standard regression ignores this → underestimates uncertainty
   Solution: time series methods (next section)
```

---

**When to Use vs When Not to Use:**

| Situation | Use Linear Regression? | Better Alternative |
|---|---|---|
| Strong, linear, causal relationship | Yes | — |
| More than 30 historical data points | Yes | — |
| Need interpretable driver relationships | Yes | — |
| Non-linear relationship visible in scatter | No | Polynomial/log regression |
| Time series with trends and seasonality | Partial | Time series decomposition |
| Short history (<20 points) | Caution | Judgment-based method |
| Complex interactions between many variables | Caution | ML methods if data rich |
| Structural market change expected | No | Scenario planning + judgment |

---

**Practical Guidance for Budgeting:**

```
Use regression for:
  □ COGS as % of revenue (very stable linear relationship)
  □ Variable overhead vs production volume
  □ Sales commissions vs revenue
  □ Variable logistics cost vs # shipments
  These relationships are physically causal and stable → regression reliable

Be cautious using regression for:
  □ Top-line revenue forecast (many non-linear factors)
  □ Any series with structural breaks in recent history
  □ Emerging business (< 2 years data)

Complement with:
  □ Management judgment: "The model says X but we know Y because..."
  □ Multiple methods: if regression and time series agree → confidence higher
  □ Scenario planning: regression provides base; management adds scenarios
```

---

**Assessing Regression Quality — quick checklist:**

```
Before trusting regression for budgeting, check:
  □ R² > 0.7 (at least 70% of variance explained)
  □ All coefficients have p < 0.05 (statistically significant)
  □ Residuals look random (no pattern in residual plot)
  □ There is a logical causal mechanism for the relationship
  □ No obvious outliers driving the relationship
  □ Forecast X values are within historical training range
  □ Recent data shows relationship still holds (not just old data)

If any check fails: acknowledge limitation; widen confidence interval;
                   seek additional methods or management judgment
```

> **Phân tích sâu:** George Box's famous quote: "All models are wrong, but some are useful." Applied to regression: a regression model of revenue will never capture all reality. But if it captures the most important drivers (GDP, pricing, store count) with reasonable accuracy, it's useful for budgeting. The goal is not a perfect model — it's a model that's better than no model or pure intuition. Track forecast errors over time; if errors are consistently in one direction, the model needs updating.

> **Sai lầm phổ biến #1:** Present regression forecast without explaining its limitations. "Our model predicts 117 tỷ." Without mentioning R² = 0.78 (22% unexplained), standard error ±9 tỷ, and the assumption that GDP forecast is accurate — the number sounds more precise than it is. Always communicate forecast uncertainty alongside the point estimate.

> **Sai lầm phổ biến #2:** Dismiss regression as "too complex" for SMEs. Simple linear regression with two variables (revenue vs one driver) takes 15 minutes to build in Excel and provides much more defensible forecast than "gut feel +15%." Even an imperfect regression is better than no quantitative structure in most budgeting decisions.

> **Cạm bẫy:** Never revisit whether the model is still valid. "We built this regression model 3 years ago." Markets change; cost structures change; business models evolve. Annual review: is R² still acceptable? Are residuals still random? Have any structural changes happened that would break the relationship? Regression is a living tool, not a one-time build.
