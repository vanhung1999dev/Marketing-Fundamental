### Khái Niệm và Mô Hình Đường Cong Học Tập

Mô hình học tập định lượng mối quan hệ giữa cumulative production volume và unit cost/time — cho phép dự báo chi phí tương lai dựa trên kế hoạch sản xuất tích lũy.

> **Bài học:** Đường cong học tập không chỉ là khái niệm định tính "chúng ta sẽ hiệu quả hơn theo thời gian." Nó là mô hình có thể quantify: "Khi cumulative production đạt 10,000 units, chi phí đơn vị sẽ là X — và chúng ta có thể dùng số này để build budget và set prices với confidence."

---

**Mô Hình Toán Học:**

```
Wright's Learning Curve Model:
  Y = a × X^b
  
  Where:
    Y = unit cost (or labour hours) for unit number X
    X = cumulative unit number (1st, 2nd, 3rd, ... nth unit)
    a = cost of first unit (Y when X=1)
    b = learning index (negative; drives the decrease)

Relationship between b and learning rate (r):
  b = log(r) / log(2)
  
  For 80% learning curve (r = 0.80):
    b = log(0.80) / log(2) = -0.09691 / 0.30103 = -0.3219
  
  For 90% learning curve (r = 0.90):
    b = log(0.90) / log(2) = -0.04576 / 0.30103 = -0.1520
```

---

**Unit Cost Calculation:**

```
80% Learning Curve Example:
  First unit cost (a) = 1,000,000 VNĐ

  Unit 1:  Y = 1,000,000 × 1^(-0.3219) = 1,000,000 VNĐ
  Unit 2:  Y = 1,000,000 × 2^(-0.3219) = 800,000 VNĐ  (80% × unit 1)
  Unit 4:  Y = 1,000,000 × 4^(-0.3219) = 640,000 VNĐ  (80% × unit 2)
  Unit 8:  Y = 1,000,000 × 8^(-0.3219) = 512,000 VNĐ  (80% × unit 4)
  Unit 16: Y = 1,000,000 × 16^(-0.3219)= 409,600 VNĐ  (80% × unit 8)
  Unit 100: Y = 1,000,000 × 100^(-0.3219) = 286,700 VNĐ
  Unit 1000: Y = 1,000,000 × 1000^(-0.3219) = 143,100 VNĐ

Excel formula: =a * X^b  or  =a * POWER(X, b)
```

---

**Cumulative Average Cost Model (alternate formulation):**

```
Some texts use cumulative average cost instead of unit cost:

YAvg = a × X^b
  YAvg = average cost of first X units (not cost of unit number X)

This is more useful for total contract/project budgeting:
  Total cost of first X units = X × YAvg = X × (a × X^b) = a × X^(1+b)

Example (80% curve, first unit = 1,000,000 VNĐ):
  Average cost of first 100 units = 1,000,000 × 100^(-0.3219) = 286,700 VNĐ
  Total cost of first 100 units = 100 × 286,700 = 28,670,000 VNĐ
  
  Compare to paying first unit cost for all 100:
  Without learning: 100 × 1,000,000 = 100,000,000 VNĐ
  With learning: 28,670,000 VNĐ = 71.3% saving due to learning!
```

**Determining Learning Curve Rate from Historical Data:**

```
If you have historical production data, estimate learning rate:

Step 1: Collect cumulative unit numbers and unit costs
  Unit #   | Cumulative | Unit Cost
  1        | 1          | 1,200,000
  10       | 10         | 680,000
  50       | 50         | 390,000
  100      | 100        | 280,000

Step 2: Take logs of both X and Y
  log(X):  0, 1, 1.699, 2
  log(Y):  6.079, 5.833, 5.591, 5.447

Step 3: Linear regression on log-log data
  log(Y) = log(a) + b × log(X)
  Slope b = estimated via SLOPE function in Excel on log-transformed data
  
  If b ≈ -0.32 → 80% learning curve
  If b ≈ -0.15 → 90% learning curve

Step 4: Verify r² > 0.8 for confidence in the estimate

Alternative (simpler):
  Find two data points where cumulative volume doubled
  Learning rate = Unit cost at doubled volume / Unit cost at original volume
```

**Learning Curve Variants:**

```
UNIT LEARNING CURVE:
  Each doubling of cumulative units → fixed % reduction in unit time/cost
  (Wright's original; most commonly used)

CUMULATIVE AVERAGE LEARNING CURVE:
  Each doubling of cumulative units → fixed % reduction in AVERAGE cost
  Results in slightly different unit cost profile
  More conservative (shows slower cost reduction)

STANFORD B MODEL:
  Similar to unit model but starts counting from unit 0 (not unit 1)
  Minor technical adjustment

PLATEAU MODEL:
  Learning curve flattens after reaching steady state
  More realistic: eventual asymptote where further learning minimal
  Important for long-run cost estimates
  Y = a × X^b + c (where c = minimum achievable cost)
```

**Budget Template Using Learning Curve:**

```
Production plan: 500 units over 12 months

Learning curve: 85% (moderate learning)
First unit cost: 2,000,000 VNĐ
b = log(0.85)/log(2) = -0.2345

Month | Cumulative | Units this | Unit cost | Monthly  | Budget
      | Production | Month      | (Yn)      | Total    | Line
------|------------|------------|-----------|----------|--------
Q1   | 50         | 50         | 1,139,000 | 56.95M   | 57M
Q2   | 150        | 100        | 880,000   | 88.0M    | 88M
Q3   | 300        | 150        | 742,000   | 111.3M   | 111M
Q4   | 500        | 200        | 653,000   | 130.6M   | 131M
Year |            | 500        |           | 386.85M  | 387M

Without learning (constant first-unit cost):
  500 units × 2,000,000 = 1,000,000,000 → overestimate by 158%!
```

> **Phân tích sâu:** In competitive bidding (especially government contracts for defense, infrastructure, complex systems), understanding the learning curve is a significant competitive advantage. A company that accurately models its learning curve can bid lower on early units (accepting lower margin), knowing future units will be profitable. A company that doesn't model it correctly either overbids (losing the contract) or underbids without realizing it (winning the contract but losing money). The learning curve is literally the difference between winning and losing in many competitive industries.

> **Sai lầm phổ biến #1:** Use average cost of current production for all future units. "Our current cost is 500,000/unit; budget is 500,000 × future volume." But current cost reflects learning from existing cumulative production. As volume grows further, cost will continue to decline. Failure to capture this → over-budget → higher prices than necessary → competitive disadvantage.

> **Sai lầm phổ biến #2:** Apply learning curve to overhead and fixed costs. The learning effect primarily applies to variable costs with significant labour content. Fixed overhead does not "learn" — it's fixed. Apply the learning model only to direct labour and variable costs; treat fixed overhead separately.

> **Cạm bẫy:** Assume learning is automatic. Learning requires: consistent production flow (no long breaks), workforce stability (low turnover), active management (documenting and sharing best practices), and systematic process improvement. Companies that assume learning just happens without actively managing it realize much slower learning than the model predicts.
