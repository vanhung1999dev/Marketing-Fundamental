### Điều Kiện Áp Dụng Đường Cong Học Tập

Biết WHEN và WHERE để áp dụng learning curve là quan trọng không kém việc biết HOW. Áp dụng learning curve vào sai ngữ cảnh dẫn đến budget sai và quyết định kinh doanh sai.

> **Bài học:** Learning curve là powerful tool trong đúng điều kiện; là misleading tool trong điều kiện sai. Checklist các điều kiện áp dụng phải được review trước mỗi lần dùng learning curve để dự báo chi phí.

---

**Điều Kiện CẦN để Learning Curve hoạt động:**

```
1. REPETITIVE TASK CONTENT:
   Tasks phải lặp lại đủ nhiều để có opportunity to learn
   
   Applies: Assembly line production; service delivery; software development
   Does NOT apply: Truly one-of-a-kind projects; research; creative work
   
   Test: "If we do this task 100 times, would we get faster/better?"
     Yes → learning curve applicable
     No → not applicable

2. SIGNIFICANT LABOUR CONTENT:
   Learning primarily reduces labour time
   If process is fully automated → no learning (machine sets the pace)
   
   High labour content: manual assembly, skilled craftsmanship, services
   Low labour content: automated manufacturing, chemical processing
   
   Test: "What % of unit cost is direct labour?"
     >30%: High potential for learning curve effect
     10–30%: Moderate potential
     <10%: Limited potential; automation dominates

3. WORKFORCE CONTINUITY:
   Same workers or teams must carry knowledge forward
   High turnover = reset of learning curve = no sustained benefit
   
   Risk factors for disrupted learning:
     >20% annual labour turnover: learning benefit partially lost
     Production gaps >3 months: skills atrophy
     Outsourcing: your learning stays with outsourcer, not you
   
   Test: "Do we have stable workforce doing this task?"

4. SUFFICIENT VOLUME:
   Learning curve math requires enough production to see the effect
   
   Minimum meaningful volume: 20+ units (ideally 100+)
   Very small batches (2–5 units): learning exists but hard to model reliably
   
   Test: "Do we produce (or plan to produce) enough units?"

5. SIMILAR PRODUCT/PROCESS:
   Learning accumulated for product A does not transfer to product B
   if they use different processes or skills
   
   Applies: Product families with shared assembly methods
   Does NOT apply: Switching to completely different technology/process
   
   Test: "Is this genuinely the same or very similar process as historical data?"
```

---

**Điều Kiện KHÔNG áp dụng learning curve:**

```
1. CAPITAL-INTENSIVE, LOW-LABOUR PROCESSES:
   Semiconductor fab, chemical plant, power generation
   Machine paces the work → human learning minimal
   → Economies of scale dominate, not learning curve

2. HIGHLY VARIABLE/CUSTOM WORK:
   Consulting projects, bespoke engineering, film production
   Each "unit" is unique → limited repetitive learning
   → Use project-based cost estimation instead

3. REGULATORY/COMPLIANCE-CONSTRAINED WORK:
   Some tasks must follow strict protocols regardless of experience
   Airline maintenance, pharmaceutical production
   → Learning in speed possible but not in effort/cost (protocol overrides)

4. COMMODITIES AND NATURAL RESOURCES:
   Mining, agriculture, oil extraction
   Cost driven by geological and market factors, not human learning
   → Input price models, not learning curve

5. AFTER PLATEAU IS REACHED:
   All learning curves flatten eventually
   For mature, high-volume products: cost improvement slows dramatically
   → Historical 80% curve may now be 98% curve
   → Check whether learning is still occurring before projecting past-rate improvement
```

---

**Practical Decision Matrix:**

| Criteria | High Learning | Low Learning |
|---|---|---|
| Labour % of cost | >30% | <10% |
| Task repetitiveness | High | Low (each unique) |
| Workforce stability | Low turnover | High turnover |
| Production volume | 100+ units/year | <20 units/year |
| Process maturity | New/growing | Fully mature |
| Automation level | Low | High |

Score each factor: total score determines whether to apply learning curve and which rate to use.

---

**Determining the Learning Rate to Use:**

```
Method 1: Empirical from own historical data (best)
  Track unit costs or times vs cumulative production
  Fit learning curve model (log-log regression)
  Read off the rate from slope

Method 2: Industry benchmark
  Aircraft manufacturing: 70–85% (Wright's original 80%)
  Electronics assembly: 80–90%
  Software development: 85–90% (per release cycle)
  Call center (AHT): 85–92%
  Construction: 90–95% (less repetitive)
  Services (professional): 90–95%

Method 3: Engineering estimate
  Estimate % reduction expected per doubling based on:
    Complexity of task
    Labour skill requirement
    Technology stability
    Workforce experience level

Caution: Use industry benchmarks only when own data unavailable;
         always validate against first few actual data points
```

---

**Conditions for Applying Learning Curve in VN SME Context:**

```
Likely applicable:
  Garment/textile manufacturer: repetitive sewing operations → strong learning
  Electronics contract assembly: repetitive assembly → strong learning
  Food processing: some operations (portioning, packaging) → moderate learning
  Software house: recurring project type → moderate learning

Likely NOT applicable:
  Trading/distribution company: no production → no learning curve
  Retail: transactions not production → minimal learning
  Professional services (consulting, legal): each case unique
  Construction company: projects differ too much (unless very standardized)

Grey area:
  Restaurant: order fulfillment has learning; menu variety limits it
  Call center: definite learning in AHT (Average Handle Time)
  Education services: some standardization → limited learning
```

> **Phân tích sâu:** One of the most misapplied concepts in business is assuming that learning curve applies everywhere. Apple's supply chain management explicitly tests: "Is this supplier on a learning curve that will reduce our component costs, or are they already at plateau?" New suppliers in early production ramp are on the steep part of the curve; mature suppliers at steady-state production are at the plateau. Different pricing strategies apply to each.

> **Sai lầm phổ biến #1:** Apply learning curve to a process that has already plateaued. If a process has been running at steady-state for 5 years, it's already at the flat part of the learning curve. Projecting additional 20% per doubling improvement is unrealistic. Check recent trend: if unit cost has been flat for last 18 months → plateau reached; don't budget for learning-curve-style cost reductions.

> **Sai lầm phổ biến #2:** Assume learning is transferable when workforce changes significantly. Rapid expansion: new factory with all new workers → learning curve resets for that facility. Budget must reflect this: initial production at new facility will be at high cost; costs decline as workforce learns. Do NOT apply existing facility's mature cost rate to new facility from day one.

> **Cạm bẫy:** Over-rely on learning curve for cost reduction; under-invest in process improvement. Learning curve shows cost falls naturally with volume. But Process improvement (lean, kaizen, 6 sigma) can dramatically accelerate the curve — or unlock improvements the natural learning curve wouldn't achieve alone. Budget for active process improvement initiatives; don't just wait for learning to happen organically.
