### Framework Chọn Công Cụ

**5 câu hỏi trước khi chọn tool:**

1. **Problem:** Tool này giải quyết problem cụ thể nào? Ai đang bị affected?
2. **Adoption:** Team có thực sự dùng nó không? Learning curve ra sao?
3. **Integration:** Nó tích hợp với những gì team đang dùng chưa?
4. **Cost:** Tổng cost (license + implementation + training + maintenance) có justify benefit không?
5. **Scalability:** Khi team lớn hơn 3×, tool này còn phù hợp không?

> **Bài học:** Câu hỏi quan trọng nhất là #2 — Adoption. Tool tốt nhất mà không ai dùng = $0 value. Tool trung bình mà mọi người dùng consistently = positive ROI. Evaluate adoption likelihood TRƯỚC khi đánh giá features.

> **Phân tích:** **Technology Acceptance Model (TAM)**: người ta adopt technology khi: (1) Perceived Usefulness — họ tin nó giúp làm việc tốt hơn, (2) Perceived Ease of Use — họ tin nó không quá khó để học. Cả hai phải đúng. Nếu tool useful nhưng phức tạp → resistance. Nếu tool dễ dùng nhưng không mang lại value → abandonment.
