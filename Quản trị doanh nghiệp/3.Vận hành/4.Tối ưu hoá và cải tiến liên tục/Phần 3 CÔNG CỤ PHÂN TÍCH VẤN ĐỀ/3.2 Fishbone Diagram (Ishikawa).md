### Fishbone Diagram (Ishikawa)

Khi vấn đề có nhiều possible causes, Fishbone Diagram giúp categorize và visualize:

```
                    [VẤN ĐỀ]
                       |
    ┌──────────────────┼──────────────────┐
    |                  |                  |
 [People]          [Process]         [Technology]
    |                  |                  |
  - Thiếu training   - SOP unclear    - Tool lỗi
  - High turnover    - No checklist   - Integration break
    |                  |                  |
 [Policy]         [Environment]      [Materials/Info]
    |                  |                  |
  - No standard      - Time pressure  - Incomplete data
  - Unclear owner    - Distractions   - Wrong format
```

> **Bài học:** Fishbone giúp tránh tunnel vision — khi nhìn vào vấn đề từ nhiều góc độ cùng lúc, team ít có khả năng bỏ sót root cause quan trọng. Sau khi brainstorm tất cả possible causes, vote để prioritize những cái nào most likely → test hypothesis → xác nhận root cause.

> **Cạm bẫy:** Dùng Fishbone để brainstorm mà không follow up bằng data/investigation để confirm root cause. Brainstorm là hypothesis generation — cần validation trước khi implement solutions.
